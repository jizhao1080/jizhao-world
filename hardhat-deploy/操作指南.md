# 寂照积分 - Hardhat部署操作指南

## 📋 前置准备

### 1. 确认Node.js已安装
```bash
node --version
npm --version
```
如果显示版本号，说明已安装。否则请先安装Node.js LTS版本。

### 2. 安装项目依赖
打开终端，进入hardhat-deploy目录：
```bash
cd "e:\禅堂（正式版）\hardhat-deploy"
"C:\Program Files\nodejs\npm.cmd" install
```

等待安装完成（可能需要几分钟）。

## 🔧 配置步骤

### 步骤1：创建.env文件
在hardhat-deploy目录下，复制模板文件：
```bash
copy .env.example .env
```

### 步骤2：编辑.env文件
用记事本或代码编辑器打开 `.env` 文件，填写以下信息：

#### ① 获取钱包私钥
1. 打开MetaMask浏览器插件
2. 切换到 **Polygon Mumbai测试网**（如果没有，需手动添加）
3. 点击账户头像 → 账户详情 → 导出私钥
4. 输入钱包密码
5. 复制显示的私钥（64位十六进制字符串，**不要带0x前缀**）

####  获取Polygonscan API Key
1. 访问 https://polygonscan.com/myapikey
2. 注册账号（如果还没有）
3. 登录后点击 "Add" 创建新的API Key
4. 复制生成的API Key

#### ③ 填写.env文件
将 `.env` 文件内容修改为：
```env
MUMBAI_RPC_URL=https://polygon-mumbai.rpc.thirdweb.com
PRIVATE_KEY=刚才复制的64位私钥
POLYGONSCAN_API_KEY=刚才复制的API Key
```

**⚠️ 注意**：
- PRIVATE_KEY **不要**包含 `0x` 前缀
- 确保没有多余的空格
- 保存文件

## 💰 获取测试币

部署合约需要MATIC测试币支付Gas费。

### 方法1：官方水龙头（推荐）
1. 访问 https://faucet.polygon.technology/
2. 选择 **Mumbai** 网络
3. 粘贴您的钱包地址（从MetaMask复制）
4. 点击 "Submit"
5. 等待1-2分钟，查看MetaMask余额是否增加

### 方法2：备选水龙头
如果官方水龙头无法使用：
- https://mumbaifaucet.com/
- https://faucet.quicknode.com/polygon/mumbai

每次可领取0.5 MATIC，足够多次部署。

## 🎯 后续操作

### 1. 添加后端服务器为铸造者
在Polygonscan的 "Write Contract" 页面：
1. 展开 `addMinter` 函数
2. 输入您的后端服务器钱包地址
3. 点击 "Write"
4. 在MetaMask中确认交易

### 2. 测试铸造积分
在Polygonscan的 "Write Contract" 页面：
1. 展开 `mintByMinutes` 函数
2. 输入：
   - recipient: 测试用户地址
   - _minutes: 30（表示30分钟禅修）
3. 点击 "Write"
4. 在MetaMask中确认交易

### 3. 查询积分余额
在Polygonscan的 "Read Contract" 页面：
1. 找到 `balanceOf` 函数
2. 输入用户地址
3. 点击 "Query"
4. 查看返回的积分数

## ❌ 常见问题排查

### 问题1：npm/npx命令找不到
**解决**：使用完整路径
```bash
"C:\Program Files\nodejs\npm.cmd" install
"C:\Program Files\nodejs\npx.cmd" hardhat run ...
```

### 问题2：部署失败 - Insufficient funds
**原因**：MATIC余额不足  
**解决**：前往水龙头领取测试币，确保余额 > 0.01 MATIC

### 问题3：部署失败 - Invalid signer
**原因**：私钥格式错误  
**解决**：
- 检查私钥是否为64位十六进制
- 确保**不包含** `0x` 前缀
- 重新从MetaMask导出私钥

### 问题4：网络连接超时
**原因**：RPC节点不稳定  
**解决**：
1. 修改 `.env` 中的 `MUMBAI_RPC_URL`
2. 更换为备用节点：`https://rpc-mumbai.maticvigil.com`
3. 重试部署

### 问题5：验证失败 - API Key无效
**解决**：
1. 检查 `.env` 中 `POLYGONSCAN_API_KEY` 是否正确
2. 前往 https://polygonscan.com/myapikey 重新生成
3. 等待几分钟后重试

### 问题6：编译错误
**解决**：
- 确保合约文件在 `contracts/` 目录
- 检查Solidity版本是否为 `^0.8.20`
- 运行 `npx hardhat compile` 单独编译测试
